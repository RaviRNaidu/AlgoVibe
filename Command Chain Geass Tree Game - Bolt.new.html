PROJECT: "Command Chain ‚Äî Geass Command Tree" (game-like web app + educational visualization)

Goal Build a single-page, fully deployable game-like website that teaches and demonstrates the "Code Geass: Lelouch's Geass Command Tree" algorithmic problem. The site must be visually polished, interactive, and include a playable mode where users build a command tree from INSERT operations and answer subtree queries.

Primary requirements (algorithm)

  1. Input format:
    • Operations: INSERT X Y (X is person id; Y is commander id or ROOT) and QUERY X.
    • First line may be N (optional) followed by N operations.
  2. Build a rooted tree with ROOT as the root (Lelouch).
  3. Compute subtree sizes (number of nodes in subtree excluding the node) and depth of each node.
  4. Answer QUERY X in O(1) after preprocessing. Batch preprocessing should be O(N).
  5. Constraints: support up to N = 100,000 in batch mode (Time limit: 2s, Memory: 256MB). Use iterative DFS for batch processing. Optionally implement Euler Tour + Fenwick tree for online inserts/queries.

UX & Game mechanics

  1. Two modes:
    • Batch mode: user pastes a full input (or uploads file), clicks Run, sees outputs and the final tree visualization.
    • Game (Placement Puzzle) mode: generate a rooted tree (level generator). Provide shuffled INSERT commands. Player must place nodes under parents (drag-and-drop or click-to-place). Validate placements against ground-truth parents.
  2. Gameplay elements:
    • Timer, Score, Combo bonus for fast correct placements.
    • Penalties (time or score) for wrong placements.
    • Levels: easy (N small), medium, hard (N larger and time-limited).
    • Queries as mini-challenges: ask player to answer QUERY X for points; correct answers show animation and explanation.
  3. Explain Mode:
    • After level completion (or on demand) animate the DFS traversal (preorder/postorder), highlight subtree calculations, and display final subtree sizes next to nodes.
    • Include step-by-step narration from the guide character.

Visual & aesthetic requirements

  1. Use D3.js for tree layout and transitions.
  2. Attractive, colorful theme ‚Äî high contrast, legible fonts, modern UI components. Provide a palette suggestion (e.g., deep navy background, purple accent, gold highlights for "Geass" effects).
  3. Character & story:
    • A guide character (original, not a trademark copy) inspired by the theme: name suggestion: "Lumen" ‚Äî charismatic strategist who teaches the player.
    • Sprite or simple SVG character with idle, point, and celebrate animations. Character gives short narrative lines as tasks progress (e.g., ‚ÄúCommand Suzaku ‚Äî watch the chain ignite!‚Äù).
  4. Animations:
    • When a node is placed, run a glowing/pulse/particle animation to indicate the command propagation.
    • Use subtle camera pan/zoom when tree grows large.
  5. Accessibility: keyboard alternatives to drag-and-drop, ARIA labels, color contrast.

# Command Chain ‚Äî Geass Command Tree

An interactive web application that teaches and demonstrates the "Code Geass: Lelouch's Geass Command Tree" algorithmic problem through gameplay and visualization.

## Overview

Command Chain is a game-like educational tool for learning tree algorithms, specifically focused on:
- Building rooted command hierarchies
- Computing subtree sizes efficiently using iterative DFS
- Answering subtree queries in O(1) time after O(N) preprocessing
- Supporting up to 100,000 nodes with optimal performance

Perfect for CS students, competitive programmers, and anyone interested in tree algorithms and data structures.

## Features

### 1. Batch Mode
Process multiple INSERT and QUERY operations at once:
- Paste or upload input files
- Fast O(N) preprocessing with O(1) queries
- Real-time tree visualization with D3.js
- Color-coded nodes by depth

### 2. Game Mode (Placement Puzzle)
Interactive learning experience:
- Three difficulty levels (Easy, Medium, Hard)
- Place nodes under correct commanders
- Score system with combo multipliers
- Timer and progress tracking
- Visual feedback for correct/incorrect placements

### 3. Explain Mode
Step-by-step algorithm visualization:
- Animated DFS traversal
- Node-by-node explanation
- Subtree size computation walkthrough
- Complexity analysis display

### 4. Guide Character (Lumen)
Charismatic strategist who provides:
- Contextual hints and tips
- Encouraging feedback
- Algorithm explanations
